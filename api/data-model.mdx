All note data is stored in the `public.notes` table. This page documents every column, the primary key, constraints, and the backing sequence.

## Table: `public.notes`

| Column | Type | Nullable | Default | Description |
|---|---|---|---|---|
| `notes_id` | `integer` | NOT NULL | `nextval('public."notes_ID_notes_seq"')` | Auto-incremented primary key |
| `notes_name` | `varchar` | NOT NULL | — | Title or name of the note |
| `notes_descr` | `text` | NOT NULL | — | Body content of the note |
| `notes_status` | `varchar` | NOT NULL | — | Completion status (`COMPLETADO` \| `INCOMPLETO`) |
| `notes_lstatus` | `varchar` | NOT NULL | — | Logical (visibility) status (`ACTIVO` \| `INACTIVO`) |

## Column reference

<ResponseField name="notes_id" type="integer" required>
  Auto-incremented primary key. Generated by the sequence `public."notes_ID_notes_seq"`. Uniquely identifies each note.
</ResponseField>

<ResponseField name="notes_name" type="varchar" required>
  The title or name of the note. Cannot be null or empty.
</ResponseField>

<ResponseField name="notes_descr" type="text" required>
  The full body content of the note. Cannot be null or empty.
</ResponseField>

<ResponseField name="notes_status" type="varchar" required>
  Indicates whether the note has been completed. Valid values:

  - `COMPLETADO` — the note is marked as done.
  - `INCOMPLETO` — the note is still pending.
</ResponseField>

<ResponseField name="notes_lstatus" type="varchar" required>
  Controls the logical visibility of the note. Valid values:

  - `ACTIVO` — the note is visible in application listings.
  - `INACTIVO` — the note is hidden; used for soft-delete.
</ResponseField>

## Primary key and constraints

| Constraint name | Type | Definition |
|---|---|---|
| `notes_pkey` | PRIMARY KEY | `notes_id` |
| `notes_ID_notes_not_null` | CHECK | `notes_id IS NOT NULL` |
| `notes_name_notes_not_null` | CHECK | `notes_name IS NOT NULL` |
| `notes_descr_notes_not_null` | CHECK | `notes_descr IS NOT NULL` |
| `notes_status_notes_not_null` | CHECK | `notes_status IS NOT NULL` |

## Sequence: `notes_ID_notes_seq`

The `notes_id` column is backed by the sequence `public."notes_ID_notes_seq"`. PostgreSQL automatically increments this sequence each time a row is inserted via `notes_agregar`. You should never set `notes_id` manually.

## Full CREATE TABLE statement

```sql
CREATE TABLE public.notes (
    notes_id     integer NOT NULL DEFAULT nextval('public."notes_ID_notes_seq"'::regclass),
    notes_name   character varying NOT NULL,
    notes_descr  text NOT NULL,
    notes_status character varying NOT NULL,
    notes_lstatus character varying NOT NULL,

    CONSTRAINT notes_pkey PRIMARY KEY (notes_id),
    CONSTRAINT "notes_ID_notes_not_null" CHECK (notes_id IS NOT NULL),
    CONSTRAINT notes_name_notes_not_null CHECK (notes_name IS NOT NULL),
    CONSTRAINT notes_descr_notes_not_null CHECK (notes_descr IS NOT NULL),
    CONSTRAINT notes_status_notes_not_null CHECK (notes_status IS NOT NULL)
);
```
